This page presents a demonstration of two examples of a contrast improvement mechanism also called a colour switcher.

Note: Both examples are not synchronized because only one of them should be used in a website.

For more information, we invite you to read the full article on this subject on Tanaguru's blog: “Text Colour contrasts (Part 3): the contrast improvement mechanism”

Explanation of the two examples:

Example 1: push-button switch with aria-pressed

Here is the detailed implementation of example 1:

  • This is a real HTML button <button type="button"></button>;
  • The button can therefore be activated with the Space bar or Enter key on the keyboard;
  • The button always has the same name, no matter what its state is. “High Contrast” is an explicit name;
  • Its change of status is indicated via:
    • The change of the value of the aria-pressed attribute on the button to ensure compatibility with assistive technologies:
      • aria-pressed="false" if you have the default choice of the feature (screen reader will say that the button is “not pressed”);
      • aria-pressed="true" if you have the secondary version of the feature, the button is pressed and stay pressed (screen reader will say that the button is “pressed”);
    • The style modification visually applied. Here, it is a pressed or not pressed button style.
  • Note: The visual change of state is not really explicit. This is the flaw in this technique.

Example 2: button with name change (without WAI-ARIA)

Here is the detailed implementation of example 2:

  • This is a real HTML button <button type="button"></button>;
  • The button can therefore be activated with the Space bar or Enter key on the keyboard;
  • The button changes its name according to its state. The name of the button indicates the action it produces:
    • “High contrast”: it applies high contrast styles;
    • “Default contrast”: it applies default styles.
  • The button name is dynamically changed via JavaScript. We will avoid having both button names in the tag <button type="button"></button> with one of them hidden via display: none; because, in case of deactivation of CSS styles, the button name will not be relevant.

Possibilities to make accessible implementations:

  • Toggle buttons / switch with:
    • either aria-pressed (example 1 on this page);
    • or button with name change (example 2 on this page).
  • ARIA Switch;
  • Radio fields;
  • Two side-by-side buttons with indication of their state (active or inactive).
