Cette page présente une démonstration de deux exemples de mécanisme d’amélioration des contrastes ou switchers de couleurs.

Note : Les deux exemples ne sont pas synchronisés car on ne doit en utiliser qu’un seul dans un site web.

Pour en savoir plus, nous vous invitons à lire l’article complet à ce sujet sur le blog de Tanaguru : « Contrastes de couleurs de texte (partie 3) : le mécanisme d’amélioration des contrastes »

Explication des deux exemples :

Exemple 1 : bouton-poussoir avec aria-pressed

Voici le détail de l’implémentation de l’exemple 1 :

  • Il s’agit d’un vrai bouton HTML <button type="button"></button> ;
  • Le bouton est donc activable avec la touche Espace ou Entrée du clavier ;
  • Le bouton a toujours le même nom, peu importe son état. « Contrastes renforcés » est un nom explicite ;
  • Son changement d’état est indiqué via :
    • Le changement de valeur de l’attribut aria-pressed présent sur le bouton afin d’assurer la compatibilité avec les technologies d’assistance :
      • aria-pressed="false" si on a le choix par défaut de la fonctionnalité (le lecteur d’écran dira que le bouton est « non-enfoncé ») ;
      • aria-pressed="true" si on a la version secondaire de la fonctionnalité, le bouton est appuyé et le reste (le lecteur d’écran dira que le bouton est « enfoncé ») ;
    • Le changement de style appliqué visuellement. Ici, il s’agit d’un style de bouton enfoncé ou non-enfoncé.
  • Note : Le changement d’état visuel n’est pas forcément très explicite. C’est le défaut de cette technique.

Exemple 2 : bouton avec changement de nom (sans WAI-ARIA)

Voici le détail de l’implémentation de l’exemple 2 :

  • Il s’agit d’un vrai bouton HTML <button type="button"></button> ;
  • Le bouton est donc activable avec la touche Espace ou Entrée du clavier ;
  • Le bouton change de nom selon son état. Le nom du bouton indique l’action qu’il produit :
    • « Contrastes renforcés » : il permet d’appliquer des styles pour avoir des contrastes renforcés ;
    • « Contrastes par défaut » : il permet de réappliquer les styles par défaut.
  • Le nom du bouton est changé dynamiquement via JavaScript. On évitera d’avoir les deux noms de boutons dans la balise <button type="button"></button> avec l’un des deux masqué via display: none; car, en cas de désactivation des styles CSS, le nom du bouton ne sera pas pertinent.

Possibilités d’implémentations accessibles :

  • Boutons toggle / switch avec :
    • soit aria-pressed (exemple 1 de cette page) ;
    • soit changement de nom du bouton (exemple 2 de cette page).
  • ARIA Switch ;
  • Champs radio ;
  • Deux boutons côte-à-côte avec indication sur leur état (actif ou inactif).
